## ✅ 验收演示流程

建议按照以下剧本进行演示，效果最佳：

### 第一幕：设备初始化与绑定 (展示安全性)
1.  **设备上电**: 按下 RST 重启设备，屏幕亮起。
2.  **未绑定状态**: 此时屏幕应显示 **「WeChat Scan」二维码**。
3.  **小程序扫码**: 
    *   打开小程序，点击首页卡片中的 **「+ 添加设备」**。
    *   进入扫码页，模拟扫码（或者手动输入 IP `192.168.145.162` 点击连接）。
4.  **绑定成功**: 
    *   手机提示“绑定成功”。
    *   **设备屏幕瞬间刷新**，进入太空人天气时钟主界面。
    *   *这是最大的亮点！体现了端云联动。*

### 第二幕：功能控制 (展示实时性)
1.  **查看设备页**: 小程序显示当前设备在线，点击进入详情。
2.  **专注番茄钟**: 
    *   在小程序底部点击 **「专注」**。
    *   选择时长（如 25 分钟），点击 **「开始专注」**。
    *   **观察设备**: 设备屏幕也会同步进入番茄钟倒计时界面（需稍等几秒同步或手动按设备切换页）。
    *   小程序点击 **「停止」**，设备端状态也会同步更新。

### 第三幕：日程同步
1.  在小程序 **「日程」** 页面添加一条新任务（如“验收答辩”）。
2.  保存后，数据通过 API 写入设备 Flash 存储。
3.  Web 端刷新页面，能看到同样的任务。

## 🛠️ 常见问题自救

*   **设备屏幕不亮？**
    *   请检查杜邦线连接，特别是 3.3V 和 GND。
    *   按一下板子上的 **RST** 复位键。
*   **小程序提示“连接超时”？**
    *   确保**手机/电脑**和**设备**连接在**同一个 WiFi** 下（spxzy666）。
    *   检查设备 IP 是否变了（查看串口监视器 `pio device monitor --baud 115200`）。
    *   确保开发者工具里勾选了「不校验合法域名」。
*   **数据已经是真实的吗？**
    *   **是的**。所有日程、专注记录、时长设置都直接读写设备内存文件（LittleFS）。
    *   小程序显示的设备信息是从设备 API 实时拉取的。
    *   天气数据由设备联网获取，如果显示 `--` 说明设备正在尝试拉取或 API 暂时繁忙，稍等片刻即可。
